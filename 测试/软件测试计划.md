## 2. 测试策略

### 2.1 整体测试策略

|  测试需求  |    参加人员    |                           预期结果                           |
| :--------: | :------------: | :----------------------------------------------------------: |
|  功能测试  | 李同学、丁同学 |                 确保用户所选定的功能正常实现                 |
|  性能测试  | 李同学、丁同学 |           测试在不同的工作条件下的性能行为响应时间           |
| 安全性测试 | 李同学、缪同学 | 对数据功能的访问正常，核实只能访问其所属用户类型被授权的功能 |
|  界面测试  | 罗同学、缪同学 | 窗口的对象和特征符合标准，进行的浏览包括窗口之间的切换，以及各种访问方法的使用 |
|  安装测试  |     刘同学     |              安装后程序可以正常进行也能正常卸载              |
|  兼容测试  |     刘同学     |               在不同的Android系统上都能够运行                |

### 2.2 进入准则

测试开始条件即进入准则： 系统已经提交，软件发布，测试计划完成; 测试用例设计完毕; 测试环境搭建完毕; 测试人员到位。    

### 2.3 暂停/退出准则

- 测试暂停条件： 软件运行时出现崩溃或发现问题、漏洞。 
- 测试退出条件： 系统功能全部测试; 缺陷严重程度为 PI 的数量为 0; 测试⽤例执⾏率= 100%; 提交测试文档。    

## 3. 测试范围和测试方法

### 3.1 测试的子系统对象

- 需要测试的子系统：

  UISubsystem子系统、UserSubsystem子系统、OnlineVSSSubsystem子系统、LocalVSSSubsystem子系统、PlayVSSSubsystem子系统、CommunicationSubsystem子系统、SensorSubsystem子系统、SensorHardwareWrapper子系统、LocalStorageSubsystem子系统、ServerSubsystem子系统

- 不需要测试的子系统：

  AudioSubsystem子系统（使用Google VR提供的API来实现空间音频播放的功能，可靠性高，无需测试）
